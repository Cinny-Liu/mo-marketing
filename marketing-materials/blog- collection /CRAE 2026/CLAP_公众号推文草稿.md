# 源自Morpheus实践的阶段性成果：MatrixOrigin论文CLAP荣获CRAE 2026最佳海报奖

近日，MatrixOrigin团队论文《CLAP: Closed-Loop Training, Evaluation, and Release Control for Domain Agent Post-training》在第三届计算机视觉、机器人与自动化工程国际学术会议（CRAE 2026）上荣获“最佳海报奖（Best Poster Award）”。

作为MatrixOrigin在Morpheus实践中持续打磨形成的一项阶段性成果，CLAP聚焦产业级Agent后训练中的训练资产治理、评测验证与发布闭环问题。此次获奖体现了团队在产业级Agent后训练、评测治理与工程化落地方向的持续探索，也反映了MatrixOrigin在“数据基础设施 + 智能体系统 + 长期记忆能力”一体化技术路线上的阶段性进展。

CRAE是聚焦计算机视觉、机器人与自动化工程的重要国际学术交流平台，长期关注智能感知、自主决策、复杂系统控制以及前沿人工智能技术在真实场景中的应用落地。此次获得最佳海报奖，不仅是对论文研究工作的认可，也说明围绕产业级Agent可靠落地所展开的方法探索，正在获得越来越多学术界与产业界的共同关注。

在大模型快速发展的今天，Agent系统正加速进入更复杂、更贴近业务核心流程的应用场景。与此同时，一个越来越现实的问题也逐渐浮出水面：

**对于企业级Agent而言，模型“训练完成”并不等于系统“可以上线”。**

真实业务环境中的数据往往伴随着字段噪声、格式不统一、标签口径变化、证据链不完整等问题。即便离线评测指标有所提升，也不意味着线上场景中的实际效果会同步改善；某些面向特定任务训练出的轻量适配器，在部分数据批次上表现更优，在另一些批次上却可能出现回退；一次训练的“成功”也未必足以支撑长期稳定的发布决策。

因此，企业真正需要的，并不仅仅是一个“能够训练”的Agent，而是一套**可训练、可评估、可验证、可发布、可回退**的完整闭环机制。

MatrixOrigin此次获奖论文CLAP，正是围绕这一问题展开。

## CLAP关注的，不是“如何把模型再调高几分”，而是“如何让Agent真正可用”

CLAP，全称为 **Closed-Loop Agent Post-training**。从定位上看，它并不是一项单纯追求算法领先的工作，也不是在提出一种替代现有微调方法的新模型架构。更准确地说，CLAP是MatrixOrigin在Morpheus实践中逐步沉淀形成的一套后训练闭环方法，聚焦Agent在真实业务场景中的训练资产治理、评测验证与发布控制。CLAP更关注一个面向真实业务场景的核心问题：

**如何把原始业务数据沉淀为可训练、可诊断、可评估、可发布的后训练资产，并结合训练完成后的离线评测、风险诊断与应用链路回放，判断一个面向特定业务任务训练出的轻量适配器，是否真正适合进入目标应用链路。**

这一问题看似务实，实则非常关键。

过去一段时间，行业对Agent能力的关注，更多集中在推理、调用工具、复杂任务分解和多轮交互等方面。但一旦进入企业环境，决定系统能否真正落地的，往往不是单点能力，而是围绕数据、训练、评测、检索、记忆和发布控制建立起来的系统能力。

从这个意义上说，CLAP所讨论的，不只是一次后训练实验，而是一套面向产业级Agent演进的治理方法。

从更长的技术路径看，Morpheus所面向的，正是产业级Agent自进化能力的建设；而CLAP则是这一方向上在Agent后训练闭环治理层面的一项重要沉淀。

## 论文的核心贡献：为领域Agent后训练建立闭环方法

围绕企业级Agent后训练中的真实难题，CLAP主要完成了四项工作。

**第一，提出了一套面向领域Agent后训练的闭环治理框架。**

CLAP将原始业务数据处理、任务材料化、训练准入、训练过程风险诊断、离线评测、应用链路回放、发布闸门与回退机制串联起来，构成一条完整的后训练控制链路。

在这一框架下，一个面向特定业务任务训练出的轻量适配器是否值得进入生产链路，不再只取决于某个单点分数是否上涨，而是由一组完整、可追踪的评测与发布证据共同决定，包括数据版本、训练配置、离线评测结果、风险记录、回归样本、应用回放结果以及时延变化等。

这使得后训练不再是“一次性实验行为”，而成为一种可验证、可审计、可复用的工程过程。

**第二，针对真实业务场景中的脏数据问题，设计了目标与证据规范化机制。**

CLAP基于匿名制造业场景数据展开研究。在这一类业务环境中，常见问题包括指标名称与数值混杂、字段定义不稳定、证据内容不可验证，以及训练集和评测集之间存在潜在重叠等。

论文针对这些问题设计了相应的数据治理与规范化策略，将原始业务记录转化为结构化SFT样本、偏好决策样本、保留评测集、风险诊断记录和发布门控依据。这一过程的价值在于，真正把“原始业务数据”沉淀成了可持续复用的后训练资产，为后续训练、评测、回归分析和发布判断提供了统一基础，而不是仅仅将其作为一次性实验输入。

**第三，基于真实实验结果，验证了后训练收益的有限性与不稳定性。**

论文在五个匿名制造业数据批次上，对QLoRA风格的LoRA-SFT进行了实验验证。结果显示，在平均意义上，轻量微调后的模型版本确实带来了一定程度的改进，包括整体得分、通过率、数值准确率和证据准确率的提升，以及幻觉率和错误事实率的下降。

但更值得关注的是，这种收益并不是在所有批次中都稳定成立。

实验表明，并非每一个批次都会同步改善；平均分数提升，也不意味着所有业务子场景都达到了安全可发布的标准；部分批次甚至会出现回退。这一结果说明，对于企业级Agent后训练而言，单一平均指标并不足以支撑上线决策，必须通过回归检测、灰度控制和发布门槛来约束风险。

**第四，通过应用链路回放验证了RAG的必要性，以及轻量适配器在真实链路中的作用边界。**

为更准确地判断后训练模型版本在真实应用中的作用边界，论文进一步设计了同一3B骨干模型上的A/B/C/D对照实验，分别比较在有无RAG、有无LoRA-SFT轻量适配器条件下的表现差异。

结论非常明确：在事实抽取任务中，RAG依然是不可替代的基础能力；这类面向特定任务训练出的轻量适配器，并不能替代检索本身，而是在检索证据存在的前提下，进一步提升结构化输出质量、核心字段匹配能力以及答案与证据之间的对齐程度。

同时，这类提升也伴随着一定的时延成本。也就是说，一个候选的轻量微调版本是否值得发布到生产环境，最终并不是“分数更高就一定更好”，而是要结合离线评测表现、风险诊断结果、应用链路回放与时延成本，进行综合判断。

这也是CLAP希望强调的核心观点：

**产业级Agent的发布，不应被视为训练完成后的自然结果，而应建立在面向真实应用链路的系统验证之上。**

## 这篇论文背后，反映的是MatrixOrigin对下一代Agent系统的整体判断

CLAP并不是一个孤立的方法研究，也不是脱离产品实践单独存在的一组抽象概念。

它背后所体现的，是MatrixOrigin对于产业级Agent建设路径的一项持续判断：未来企业级智能体的竞争力，不会仅仅来自更大的模型或更多的参数，而将越来越取决于围绕模型构建起来的系统能力，包括数据基础设施、长期记忆体系、检索增强能力、应用验证机制以及可控发布流程。此次论文获奖，更可以被视为Morpheus在后训练工程方法上的一次阶段性成果展示。

围绕这一方向，MatrixOrigin正在持续推进多个核心项目协同演进，逐步形成面向产业级Agent的产品矩阵。

**Morpheus**可以理解为这套体系中的训练资产与演进治理层，也是Agent自进化能力的重要承载体。它面向复杂业务场景中的智能体持续演进展开建设，重点不是简单“堆数据做微调”，而是把真实业务数据、反馈信号、评测结果和应用表现持续沉淀为可治理的后训练资产，包括训练样本、偏好样本、保留评测集、门控信号、诊断记录与实验结论等，并围绕这些资产建立从数据构造、训练验证、效果评估到发布判断的闭环。某种意义上，Morpheus关注的是Agent能力本身如何在业务环境中持续学习、持续修正、持续增强，这正是自进化最核心的一层。

**Astra**是面向工程落地的Agent平台，更接近运行时与编排层，但它同样具有自进化成分。它强调会话可审计、状态可管理、过程可回放、能力可治理，并支持围绕Agent运行过程建立触发、验证、回放和门控机制。无论是训练后的效果验证、关键流程中的自动触发，还是面向高风险动作的人机协同审核，Astra都承担着把模型能力接入真实业务链路、并让整条链路保持可观测、可复盘、可控制的重要角色。如果说Morpheus更关注“能力如何进化”，那么Astra更关注“运行中的系统如何根据反馈持续优化”，两者共同构成了Agent自进化闭环中离线与在线的两个关键层面。

**Memoria**则聚焦Agent长期记忆体系建设，围绕持久化记忆、检索增强、图谱化关联、治理与回退等关键能力，支撑智能体在多轮、多任务、长周期环境中保持持续可用的上下文能力。更进一步，Memoria还引入了类似 Git-for-Data 的能力，包括 snapshot、branch、merge、diff、rollback 等机制，使记忆与数据状态不再只是“被存下来”，而是能够像代码一样被版本化、比较、回退和治理。这为Agent系统的长期演进、问题追踪和安全发布提供了更强的基础设施支撑。

从训练资产沉淀，到运行过程中的自动触发与验证，再到记忆、数据与状态的版本化治理，这些项目并不是彼此孤立的点状能力，而是在共同构建一条真正面向企业场景的Agent自进化闭环。

如果把这几项能力放在一起看，可以更清晰地理解CLAP的现实意义。它并不只是论文中的一个方法名称，更代表了一种完整的系统思路：

**以训练资产和数据基础设施驱动Agent能力自进化，以自动触发、验证与回放机制推动运行链路持续优化，以长期记忆和版本化治理能力支撑状态演进，并以评测与发布闭环保障Agent真正可用。**

这也是为什么，此次论文获奖对MatrixOrigin而言，不只是一次学术成果展示，更是公司在产业级Agent技术路线上的阶段性验证。

## 从“能做出来”到“能稳定上线”，产业级Agent正在进入深水区

随着大模型和Agent技术不断成熟，行业关注点正在发生变化。

过去，大家更关注Agent“能不能做出来”；现在，真正关键的问题已经变成Agent“能不能稳定上线、能不能持续演进、能不能进入业务核心流程并长期承担任务”。

这是产业级Agent从能力探索走向基础设施建设的关键阶段。

在这个阶段里，数据质量、训练边界、检索依赖、记忆机制、证据对齐、链路回放、发布门槛和回退机制，都会成为决定系统成败的关键因素。它们或许不像单点Demo那样直观，却直接影响Agent是否真正可信、可控、可用。

CLAP此次在CRAE 2026获得最佳海报奖，正说明围绕这些关键问题的系统性研究，正在得到更广泛的认可。

MatrixOrigin也将继续沿着这一方向推进相关能力的产品化与平台化建设，持续完善面向产业级Agent的数据、记忆、检索、评测和发布体系，推动智能体从“可以试用”走向“可以信任”，从“能够回答”走向“能够承担真实任务”。

因为我们相信，下一代企业智能体的核心竞争力，不只是模型能力本身，更是支撑模型进入真实世界的工程体系与基础设施能力。
